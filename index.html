Understanding how absolute positioning works in relation to its nearest positioned ancestor. This is crucial because it teaches you how to create a positioning context, which is the foundation for building complex layouts with layered elements.

You have a sage parent container with a yellow child box inside it. Your goal is to position the yellow box in the bottom-right corner of the sage container, exactly 20 pixels from the bottom edge and 20 pixels from the right edge. The yellow box should be completely removed from the normal document flow.

Your Task:

Make the yellow box appear in the bottom-right corner of its sage parent container. The box should be positioned exactly 20 pixels away from both the bottom and right edges of the sage container. Other content in the container should flow as if the yellow box does not exist at all.

You need two pieces here. First, the parent container needs to become a positioning reference point by giving it position: relative. Then, the child needs position: absolute so it positions itself relative to the parent. Use bottom and right properties to push it into the corner.

.ex2-parent {
  position: relative;

.ex2-child {

CSS you would write:

.ex4-header {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  z-index: 100; /* Ensures it appears above other content */
}

.ex4-content {

}

Why this works: Fixed positioning attaches the element to the viewport itself, not the document. When you scroll, the viewport does not move relative to fixed elements, so they appear to stay in place. The z-index ensures the header appears on top of scrolling content, and the padding-top on content prevents overlap.

Building Sticky Section Headers

ðŸŽ¯ What You'll Learn:

Mastering sticky positioning, which is one of the most useful and modern positioning techniques. Sticky elements behave like relative positioning until they reach a scroll threshold, then they stick in place like fixed elements. This is perfect for section headers that should stick when scrolled into view.

You have multiple content sections, each with a colored heading. Your challenge is to make each section heading stick to the top of the scrollable area when the user scrolls to that section. As you scroll down past "Section 1", its heading should stick at the top. When you reach "Section 2", its heading should push the previous one out of the way and stick instead. This creates a beautiful layered scrolling effect.

Your Task:

Apply sticky positioning to the section headings so they stick to the top of the scroll container when scrolled to. Each heading should stick at top: 0px of its scroll container. When a new section heading reaches the top, it should stack on top of or push away the previous sticky heading. Try scrolling through all sections to see the effect working properly.

The key is applying position: sticky to the heading elements along with a top value that tells them when to stick. Think of sticky as a hybrid between relative and fixed. CSS you would write:

.ex5-heading {
  position: sticky;
  top: 0;
  z-index: 10; /* Ensures headings stack properly */
}

Why this works: Sticky positioning makes elements behave like relatively positioned until they scroll to the threshold specified by top. Once they reach top: 0, they become fixed at that position. When the next sticky element arrives, it pushes the previous one out of the way. This creates the beautiful layered scrolling effect you see in modern apps and websites. The z-index ensures proper stacking order when multiple sticky elements interact.

Building a Modal Overlay
Hard

ðŸŽ¯ What You'll Learn:

Combining fixed positioning with absolute positioning to create a modal overlay that covers the entire viewport with a darkened background while centering a content box. This is one of the most common patterns in web development and teaches you how to layer elements and center content using positioning techniques.

You have an image gallery where clicking an image should open a full-screen modal overlay. The overlay should cover the entire viewport with a dark semi-transparent background, and a white content box should appear perfectly centered in the middle of the screen. This is exactly how lightboxes, dialog boxes, and modals work on real websites.

Your Task:

Create a modal overlay that appears when clicking any image in the gallery. The overlay should use fixed positioning to cover the entire viewport from edge to edge. The white content box inside should be centered both horizontally and vertically. You will need to use a combination of positioning techniques along with transform to achieve perfect centering. The overlay should have a dark background with some transparency.

The overlay background needs to cover the entire viewport, so use position: fixed with top: 0, left: 0, right: 0, and bottom: 0. This makes it stretch to fill the whole screen. Give it a dark background color with transparency like background: rgba(0, 0, 0, 0.9).

ðŸ’¡ Click for Hint #2 - Centering the Content

To center the white content box, you can use absolute positioning combined with a transform trick. Position the box absolutely with top: 50% and left: 50%, which moves its top-left corner to the center of the overlay. Then use transform: translate(-50%, -50%) to shift it back by half its own width and height, achieving perfect centering.

CSS you would write:

.ex6-overlay {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  bottom: 0;
  background: rgba(0, 0, 0, 0.9);
  z-index: 1000; /* Appears above everything */
}

.ex6-overlay-content {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}

Why this works: Fixed positioning on the overlay makes it cover the viewport regardless of scroll position. Setting all four edges to 0 stretches it to fill the screen. The content box uses absolute positioning (which positions it relative to the fixed overlay) with top: 50% and left: 50% to move its corner to the center. The transform then shifts it back by half its dimensions, achieving perfect centering. This is the standard pattern for modals, lightboxes, and dialog overlays in modern web development.
